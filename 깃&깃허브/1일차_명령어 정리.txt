1. 깃 초기화 -> 깃 저장소 생성
$ git init

2. 깃 상태 확인 
$ git status 

3. 파일 스테이지에 올리기 -> 스테이징
$ git add 파일명
$ git add . -> 전체 파일 스테이징

4. 커밋한다 -> 버전으로 생성한다
$ git commit -m "커밋메시지"

5. 로그 확인 -> 작성자, 날짜, 시간, 커밋메시지
$ git log


< 작업 되돌리기 >
1. 작업 트리 수정한 내용 되돌리기
2. 스테이징 취소하기
3. 최신 커밋 취소하기 -> git reset HEAD^


< 특정 커밋으로 되돌리기 >
1. reset 
- 혼자 사용할 때 사용
- 커밋 기록을 깔끔하게 유지 가능
  v5 -> v2 : v3, v4, v5 모두 기록에서 제거

2. revert
- 팀 프로젝트 시, 사용
- 커밋을 취소하더라도 커밋 기록을 유지
  v5 -> v2 : v3, v4, v5 기록에 유지
